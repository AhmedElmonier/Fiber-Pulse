## Summary
Build a production-grade Telegram Bot interface that serves as the primary delivery vehicle for FiberPulse AI. This includes command-based forecasting, scheduled market pulses (4x daily Cairo time), real-time volatility alerts, and visual chart generation (Fan/Bar charts) to provide operational clarity to whitelisted stakeholders.

## Technical Context

**Language/Version**: Python 3.12  
**Primary Dependencies**: `python-telegram-bot` (v20+), `matplotlib`, `APScheduler`, `SQLAlchemy`, `asyncpg`  
**Storage**: PostgreSQL with `pgvector` (existing)  
**Testing**: `pytest`  
**Target Platform**: Linux server  
**Project Type**: Telegram Bot / Web Service  
**Performance Goals**: Bot command response < 2s; Reports delivered within 5m of target.  
**Constraints**: Africa/Cairo timezone for all scheduling; whitelisted access only.  
**Scale/Scope**: <100 whitelisted users; 4x daily scheduled reports; real-time per-instrument alerts.
